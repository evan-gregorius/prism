# AI-Assisted Development Statement

PRISM uses AI-assisted tools to accelerate parts of the development workflow, including research discovery and organization, documentation drafting, code scaffolding, debugging support, test generation, refactoring suggestions, and interface prototyping.

The human developer remains responsible for selecting the technical direction, choosing and checking sources, defining the architecture, reviewing generated code, checking equations and units, designing validation tests, interpreting results, rejecting unsupported claims, and approving public releases.

AI-generated citations, technical claims, equations, and code are not treated as correct without review.

Consequential AI-assisted decisions or code should be traceable through the engineering notebook, commit history, source records, tests, and review notes. The project does not attempt to record every routine prompt or minor interaction.
